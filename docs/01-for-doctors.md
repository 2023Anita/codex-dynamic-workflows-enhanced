# For Doctors And Clinical Educators

This workflow helps clinicians use AI without hiding the parts that still require professional judgment.

It is useful for:

- literature review planning
- teaching-session outlines
- evidence tables
- patient education drafts
- medical AI project documentation
- repository or manuscript preparation

## What The Workflow Adds

AI can summarize quickly, but medical work needs explicit boundaries:

- What clinical question is being asked?
- Which sources were searched?
- Which claims are supported by citations?
- Which claims are uncertain?
- What requires clinician review?
- What should not be treated as a guideline or diagnosis?

Dynamic workflows keep these questions visible.

## Recommended Medical Work Packets

```text
1. Define clinical question and audience.
2. Set inclusion, exclusion, and source date boundaries.
3. Build evidence table.
4. Draft teaching or education outline.
5. Verify citations and clinical safety language.
6. Mark final output for clinician review.
```

## Approval Gates

Ask for human confirmation before:

- finalizing a clinical question
- relying on a guideline or recommendation
- publishing patient-facing content
- sending material to learners, patients, or colleagues
- claiming that evidence is current or definitive

## Safety Language

Use language such as:

```text
This is educational support and does not replace clinician judgment, local policy, or current clinical guidelines.
```

Do not use language that implies the AI output is a diagnosis, treatment plan, or current guideline.
