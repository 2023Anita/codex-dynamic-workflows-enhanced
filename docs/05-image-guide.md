# Image Guide

The images in `assets/` are designed for README and teaching use.

## `workflow-overview.png`

Purpose: show the full dynamic workflow.

Use it when introducing the project to a general audience.

## `packet-gate-verification-loop.png`

Purpose: explain the core mechanism.

Use it when teaching why the agent should not automatically continue through risky steps.

## `doctor-literature-review.png`

Purpose: show a clinician-facing research workflow.

Use it when talking to doctors, educators, or medical AI teams.

## `patient-education-workflow.png`

Purpose: show safe patient-facing drafting.

Use it when explaining red flags, boundaries, plain language, and clinician review.

## `beginner-github-workflow.png`

Purpose: show GitHub publishing for beginners.

Use it when explaining local files, commits, pushes, and public release.

## `doctor-beginner-bridge.png`

Purpose: show the workflow architecture behind the teaching examples.

Use it when explaining that a workflow is more than a prompt: it needs routing, stage contracts, state files, validators, artifacts, and human confirmation.
